Guava#Splitter 

三月 12, 2023 铁柱
common develop guava java

简介

Guava是Google提供的一套Java核心库，其中的Splitter类是用于字符串分割的工具类。

学习目标

本文将从以下几个方面介绍Splitter的使用方法和注意事项：

  • Splitter的基本用法
  • Splitter的高级用法
  • Splitter的性能优化
  • Splitter的注意事项

Splitter的基本用法

  1. 导入Guava库

    在项目中引入Guava库的依赖，例如Maven配置文件中添加以下代码：

    <!-- https://mvnrepository.com/artifact/com.google.guava/guava -->
    <dependency>
      <groupId>com.google.guava</groupId>
      <artifactId>guava</artifactId>
      <version>31.1-jre</version>
    </dependency>

  2. 创建Splitter对象

    使用Splitter类提供的静态方法创建Splitter对象，例如：

    Splitter splitter = Splitter.on(",");

    上述代码创建了一个以逗号为分隔符的Splitter对象。

  3. 使用Splitter对象分割字符串

    调用Splitter对象的split()方法对字符串进行分割，例如：

    Iterable<String> result = splitter.split("a,b,c");

    上述代码将字符串”a,b,c”按照逗号分割，并返回一个包含分割后子串的可迭代对象。

Splitter的高级用法

  1. 指定分隔符

    除了使用on()方法指定分隔符外，还可以使用onPattern()方法指定正则表达式作为分隔符，例如：

    Splitter splitter = Splitter.onPattern("\\\\s+");

    上述代码创建了一个以空白字符为分隔符的Splitter对象。

  2. 去除分割后子串的前后空白字符

    使用trimResults()方法可以去除分割后子串的前后空白字符，例如：

    Splitter splitter = Splitter.on(",").trimResults();

    上述代码创建了一个以逗号为分隔符，并去除分割后子串的前后空白字符的Splitter对象。

  3. 过滤空串

    使用omitEmptyStrings()方法可以过滤分割后的空串，例如：

    Splitter splitter = Splitter.on(",").omitEmptyStrings();

    上述代码创建了一个以逗号为分隔符，并过滤分割后的空串的Splitter对象。

  4. limit分割

    使用limit() 定义分割size()

    /**
   * Split - limit
   */
  @Test
  public void testSplitOnSplit_limit() {
    List<String> strings = Splitter.on("|").limit(3).trimResults()
        .splitToList("我|你|她|他");
    Assert.isTrue(strings.size() == 3 && Objects.equals(strings.get(2), "她|他"));
  }

  5. 将字符转为Map

    将字符串转为Map，可以使用withKeyValueSeparator()方法。示例代码如下：

    /**
   * Split - toMap
   */
  @Test
  public void testSplitOnSplit_ToMap() {
    Map<String, String> split = Splitter.on(";").withKeyValueSeparator('=')
        .split("name=Tom;age=12");
    Assert.isTrue(split.size() == 2 && Objects.equals(split.get("name"), "Tom"));
  }

    上述代码将字符串”name=Tom;age=12”按照分号分割，并将每个子串按照等号分割为键值对，最终返回一个包含键值对的Map对象。

  6. 自定义Splitter对象

    使用Splitter.Builder()方法可以创建自定义的Splitter对象，例如：

    Splitter splitter = new Splitter.Builder()
                        .trimResults()
                        .omitEmptyStrings()
                        .splitter(",");

    上述代码创建了一个以逗号为分隔符，并去除分割后子串的前后空白字符和过滤分割后的空串的自定义Splitter对象。

Splitter的性能优化

  1. 静态Splitter对象

    由于Splitter对象是线程安全的，因此可以将其声明为静态对象，避免反复创建对象的开销，例如：

    private static final Splitter SPLITTER = Splitter.on(",");

  2. 使用splitToList()

    对于需要多次分割字符串的场景，使用splitToList()方法可以避免每次都创建可迭代对象的开销，例如：

    List<String> result = splitter.splitToList("a,b,c");

Splitter的注意事项

  1. Splitter对象不可变

    Splitter对象是不可变的，一旦创建就不能修改其属性，例如：

    splitter.omitEmptyStrings();  // 错误示例

    上述代码将会抛出异常。

  2. Splitter对象是线程安全的

    Splitter对象是线程安全的，可以在多线程环境下使用。

  3. Splitter对象不支持null值

    Splitter对象不支持null值，调用split(null)方法将会抛出异常。
