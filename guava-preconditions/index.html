Guava Preconditions API 是 Google Guava 库中提供的一组用于参数校验的工具类。本文将介绍该 API 的学习大纲，包括：

  1. 为什么要使用 Preconditions API

  2. Preconditions API 的基本用法

  3. Preconditions API 的高级用法

  4. 注意事项和常见问题

为什么要使用 Preconditions API

在编写复杂的代码时，参数校验是一个必不可少的环节。如果没有对参数进行校验，可能会导致代码运行时出现各种异常，甚至是安全漏洞。使用 Guava Preconditions API 可以使参数校验变得更加简单和易于维护。

Preconditions API 的基本用法

Guava Preconditions API 提供了多种方法来校验参数，包括：

  • checkArgument(boolean expression)：

    校验表达式是否为 true，如果为 false，抛出 IllegalArgumentException 异常。

  • checkNotNull(T reference):

    校验对象是否为 null，如果为 null，抛出 NullPointerException 异常。

  • checkState(boolean expression)：

    校验对象的状态是否正确，如果为 false，抛出 IllegalStateException 异常。

  • checkElementIndex(int index, int size)：

    校验下标是否越界，如果越界，抛出 IndexOutOfBoundsException 异常。

  • checkPositionIndex(int index, int size)：

    校验下标是否为非负数且不大于 size，如果不符合要求，抛出 IndexOutOfBoundsException 异常。

  • checkPositionIndexes(int start, int end, int size)：

    校验起始下标和结束下标是否合法，如果不合法，抛出 IndexOutOfBoundsException 异常。

以下是一个使用 Preconditions API 的示例：

import com.google.common.base.Preconditions;
​
public class Example {
  public void doSomething(String name, int age) {
    Preconditions.checkNotNull(name, "Name must not be null");
    Preconditions.checkArgument(age >= 0 && age <= 150, "Age must be between 0 and 150");
​
    // 正常业务逻辑
  }
}

Preconditions API 的高级用法

除了基本用法之外，Guava Preconditions API 还提供了一些高级用法，包括：

  • 自定义异常信息格式：可以使用 %s、%d、%f 等占位符来格式化异常信息。

  • 自定义异常类型：可以使用 checkArgument、checkState 等方法的重载版本，自定义异常类型。

  • 异常信息的延迟计算：如果异常信息的计算比较耗时，可以使用 Supplier 接口来延迟计算，避免不必要的开销。

以下是一个使用高级用法的示例：

import com.google.common.base.Preconditions;
​
public class Example {
  public void doSomething(String name, int age) {
    Preconditions.checkNotNull(name, "Name must not be null");
    Preconditions.checkArgument(age >= 0 && age <= 150, "Invalid age: %s", age);
​
    // 正常业务逻辑
  }
}

注意事项和常见问题

在使用 Guava Preconditions API 时，需要注意以下几点：

  • 不要滥用 Preconditions：Preconditions 适用于简单的参数校验场景，如果校验逻辑比较复杂，建议使用其他方式进行校验。

  • 不要滥用异常：异常应该只用于处理异常情况，不应该用于控制程序流程。

  • 不要忘记异常信息：在抛出异常时，一定要提供明确的异常信息。
