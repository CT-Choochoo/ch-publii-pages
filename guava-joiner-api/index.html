Guava#Joiner

三月 11, 2023 by 铁柱

介绍

  • 什么是 Guava Joiner API

Guava Joiner API 是 Google Guava 库中的一个字符串连接工具。它可以用来方便地将多个字符串或集合类拼接成一个完整的字符串，支持自定义连接符、跳过 null 值等功能。

  • Guava Joiner 是线程安全的么

Guava Joiner 是线程安全的。Guava 官方文档明确说明了 Joiner 实例是不可变的，因此可以安全地在多个线程之间共享和重用。

在使用 Joiner 的过程中，如果需要在多个线程之间共享一个 Joiner 实例，可以将其声明为静态 final 变量，以确保在多个线程之间共享时的线程安全性。

需要注意的是，如果对 Joiner 实例进行修改，例如使用 withXXX() 方法修改 Joiner 的行为，则 Joiner 实例将不再是不可变的，此时需要采取适当的同步措施来确保线程安全性。

进阶

  • 使用 Joiner 格式化输出

使用 Joiner 格式化输出

可以使用 Joiner 来格式化输出，比如输出成表格形式:

private final List names = Arrays.asList("Tom", "Jerry", "Peter", "Mary");
private final List ages = Arrays.asList(18, 20, 25, 30);

@Test
public void testJoiner_Output_Formatting() {
// 创建一个二维列表
List> table = new ArrayList<>();
// 添加表头
table.add(Arrays.asList("Name", "Age"));
// 添加数据
for (int i = 0; i < names.size(); i++) {
table.add(Arrays.asList(names.get(i), ages.get(i).toString()));
}
// 使用 joiner 将二维列表拼接成字符串
String join = Joiner.on("\\n").join(
table.stream().map(input -> Joiner.on("\\t").join(input)).collect(Collectors.toList()));

// 断言输出结果是否符合预期
Assert.equals(join, "Name\\tAge\\nTom\\t18\\nJerry\\t20\\nPeter\\t25\\nMary\\t30");
}

这样就可以将一个二维列表拼接成表格形式的字符串输出。

  • 使用 Joiner 扩展实现自定义的 Joiner
  • 使用 Joiner 扩展实现自定义的 Joiner

Guava Joiner API 提供了一些常用的连接符和连接方式，如果我们需要使用一些特殊的连接符或者连接方式，可以通过 Joiner 提供的静态方法创建一个新的 Joiner 实例。也可以通过继承 Joiner 类并覆写其中的方法来实现自定义的 Joiner。

public class MyJoiner extends Joiner {

private final String separator;

private MyJoiner(String separator) {
this.separator = separator;
}

public static MyJoiner on(String separator) {
return new MyJoiner(separator);
}

@Override
public A appendTo(A appendable, Iterable parts) throws IOException {
Iterator iterator = parts.iterator();
if (iterator.hasNext()) {
appendable.append(toString(iterator.next()));
while (iterator.hasNext()) {
appendable.append(separator);
appendable.append(toString(iterator.next()));
}
}
return appendable;
}

@Override
public A appendTo(A appendable, Object[] parts) throws IOException {
return appendTo(appendable, Arrays.asList(parts));
}

@Override
public StringBuilder appendTo(StringBuilder builder, Iterable parts) {
try {
return super.appendTo(builder, parts);
} catch (IOException impossible) {
throw new AssertionError(impossible);
}
}

@Override
public StringBuilder appendTo(StringBuilder builder, Object[] parts) {
return appendTo(builder, Arrays.asList(parts));
}
}

上面的代码中，我们定义了一个新的 MyJoiner 类，继承了 Joiner 类，并重写了其中的几个方法。在 appendTo 方法中，我们使用了一个迭代器来遍历字符串列表，并将其拼接成一个完整的字符串。另外，我们还可以在 MyJoiner 类中添加一些自定义的方法，来扩展 Joiner 的功能。

@Test
public void testJoiner_Custom_Joiner() {
// 使用自定义的 Joiner
String join = MyJoiner.on("|").join(Arrays.asList("A", "B", "C"));
Assert.equals(join, "A|B|C");
}

这样我们就可以使用自定义的 Joiner 来进行字符串拼接，实现更加灵活的功能。

总结

  • Guava Joiner API 的优缺点

优点：

  1. 简单易用：Guava Joiner 提供了一种简单易用的方式将多个字符串或对象连接起来，并且可以指定分隔符、前缀、后缀等连接参数。
  2. 可配置性强：Guava Joiner 可以通过链式调用 withXXX() 方法来配置连接参数，例如设置分隔符、跳过 null 元素、添加前缀和后缀等，非常灵活。
  3. 性能高效：Guava Joiner 在内部使用 StringBuilder 进行字符串拼接，并且使用 append() 方法来连接字符串，这种方式比使用 + 连接字符串要更加高效。
  4. 支持多种数据类型：Guava Joiner 不仅支持连接字符串，还支持连接各种数据类型，包括数组、集合、迭代器等。

缺点：

  1. 可能会增加代码复杂性：在一些简单的场景下，使用 Guava Joiner 可能会增加代码的复杂性，例如连接简单的字符串时。
  2. 不支持自定义格式：Guava Joiner 可以指定分隔符、前缀和后缀等连接参数，但是不支持自定义连接格式，例如指定每个元素的宽度或精度。
  3. 可能会产生不可变对象：Guava Joiner 返回的结果是一个不可变的字符串，因此如果需要修改连接后的字符串，需要创建一个新的对象，这可能会增加内存开销。
  • Guava Joiner API 与其他连接字符串的比较
  • 与 StringBuilder 相比，Guava Joiner API 使用更加方便，可以支持多种数据类型的拼接，但在性能方面可能略逊一筹。
  • 与 Apache Commons Lang 库中的 StringUtils 相比，Guava Joiner API 支持更多的数据类型，但 StringUtils 在性能方面可能更优秀。
  • Guava Joiner API 的应用场景
  • 字符串拼接：Guava Joiner API 可以用来方便地将多个字符串拼接成一个完整的字符串。
  • 数据输出：可以使用 Guava Joiner API 将数据按照指定的格式进行输出，比如输出成表格形式。
  • 数据导出：在数据导出时，可以使用 Guava Joiner API 将多个数据项拼接成一行并输出到文件中。
